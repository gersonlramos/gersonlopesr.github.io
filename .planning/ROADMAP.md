# Roadmap: Portfólio — Cientista de Dados & Engenheiro de Dados

## Overview

The journey goes from an empty repository to a live, polished portfolio site at a real GitHub Pages URL. Phase 1 stands up the technical foundation (Astro + MDX + Tailwind, CI/CD to GitHub Pages, base layout) so every later phase has a real, verifiable deploy target. Phase 2 establishes the site's narrative — who this hybrid Data Scientist/Data Engineer is, their experience, and how to reach them — before any case study is drafted, since the positioning should inform which projects get featured. Phase 3 delivers the core value of the entire project: a locked problem→approach→impact case-study template applied to the user's real DS and DE projects, with business-impact framing, architecture diagrams, and a flagship interactive demo. Phase 4 hardens everything that's easy to overlook once content exists — performance, mobile responsiveness, SEO, and a full link-check — before calling v1 launched.

## Phase Details

### Phase 1: Foundation & Deployment Pipeline
**Goal**: The site's technical foundation is live at a real, working GitHub Pages URL, with every subsequent phase able to deploy and verify against it.
**Depends on**: Nothing (first phase)
**Requirements**: SETUP-01, SETUP-02, SETUP-03
**Success Criteria** (what must be TRUE):
  1. The site is built with Astro + MDX + Tailwind and automatically redeploys via GitHub Actions whenever changes are pushed
  2. The site is reachable at a working `username.github.io` URL with no broken paths or 404s
  3. Every page shows a persistent navigation bar and footer containing contact/profile links

### Phase 3: Case Study Template & Project Content
**Goal**: A visitor can browse the user's real projects and read case studies that clearly communicate technical capability and the reasoning behind decisions — the core value of the site.
**Depends on**: Phase 2
**Requirements**: PROJ-01, PROJ-02, PROJ-03, PROJ-04, PROJ-05, PROJ-06
**Success Criteria** (what must be TRUE):
  1. Visitor browses a gallery/list of all projects with visible technology tags
  2. Visitor opens a dedicated case study page for each project, following a problem→approach→impact template, that links out to the full code/notebook on GitHub instead of embedding it
  3. Each case study explicitly states the business/practical impact of the project, not just a technical metric
  4. Data Engineering case studies include a static architecture/pipeline diagram
  5. At least one flagship project links to an externally-hosted interactive demo

### Phase 4: Polish, Performance & Launch
**Goal**: The site is production-ready — fast, responsive, discoverable, and fully verified — and v1 is considered launched.
**Depends on**: Phase 3
**Requirements**: POLISH-01, POLISH-02, POLISH-03, POLISH-04

  2. Images are compressed and correctly sized so pages load quickly
  3. Every link on the site has been checked and confirmed working (no dead links)
  4. Site includes SEO meta tags (meta description, Open Graph, sitemap) verifiable via page source
