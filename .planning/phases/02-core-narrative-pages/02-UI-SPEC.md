# Phase 2 - UI Design Contract

## Scope

Narrative-first homepage sections and supporting navigation/contact behavior:

- About/Hero
- Experience summary (CV style)
- Resume download action
- Contact/profile link completeness

## Visual Continuity

- Reuse Phase 1 tokens and typography roles.
- Keep dark-tech visual direction unchanged.
- Preserve persistent Nav/Footer shell.

## Copy Contract

- Hero headline must state hybrid role: Data Scientist + Data Engineer.
- About paragraph must explain focus: decision reasoning + business impact.
- Experience section must present concise chronology and outcomes.
- Resume CTA copy should be explicit: "Download Resume (PDF)".

## Accessibility

- Maintain keyboard-visible focus styles on all links/buttons.
- Ensure section headings follow semantic order (`h1` then `h2`).
- Keep icon-only footer links labeled for assistive technologies.

## Mobile Behavior

- Sections stack naturally with readable line length.
- Experience cards/entries remain legible on small screens.
- Resume CTA remains visible above first fold on common mobile widths.

## Non-Goals

- Complex animations/interactions.
- New visual theme direction.
- Case-study gallery/cards (Phase 3).
