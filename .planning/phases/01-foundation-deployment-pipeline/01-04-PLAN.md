After pushing, poll the GitHub Actions run status via the public REST API (no auth token needed for a public repo's Actions runs list) every 15 seconds, up to 5 minutes:

```bash
for i in $(seq 1 20); do
  STATUS=$(curl -s https://api.github.com/repos/gersonlopesr/gersonlopesr.github.io/actions/runs?branch=main\&per_page=1 | grep -o '"status":"[a-z_]*"' | head -1)
  CONCLUSION=$(curl -s https://api.github.com/repos/gersonlopesr/gersonlopesr.github.io/actions/runs?branch=main\&per_page=1 | grep -o '"conclusion":"[a-z_]*"' | head -1)
  echo "Attempt $i: $STATUS $CONCLUSION"
  if echo "$STATUS" | grep -q "completed"; then break; fi
  sleep 15
done
```

The workflow run is successful when `STATUS` shows `"status":"completed"` and `CONCLUSION` shows `"conclusion":"success"`. If it shows `"conclusion":"failure"`, fetch the run's job logs URL from the same API response (`html_url` field) and report the failure to the user before proceeding — do not mark this task done on a failed run.
