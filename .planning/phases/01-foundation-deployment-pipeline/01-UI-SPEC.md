# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Phase Scope Note

Phase 1 delivers the technical shell only: persistent `Nav` + `Footer` (SETUP-03), a placeholder home page, and (recommended, low-cost) a custom 404 page so the shared layout is exercised on an error path too. It does **not** deliver About/Experience/Contact content (Phase 2) or the Projects gallery/case studies (Phase 3). This contract governs the shell components and establishes tokens (spacing, type, color) that Phases 2–4 must reuse without redefinition.

No existing design system was found (`components.json`, `tailwind.config.*`, `src/` all absent — confirmed via filesystem scan, project is 100% greenfield per CONTEXT.md D-03). All values below are new declarations, not confirmations of existing code.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-rolled `.astro` components (`BaseLayout.astro`, `Nav.astro`, `Footer.astro` per 01-RESEARCH.md Architecture Patterns) |
| Icon library | `lucide-astro` (tree-shaken, zero runtime JS — matches project's zero-JS-by-default philosophy; used for footer social-link icons) |
| Font | Primary: **Inter** (variable, self-hosted via `@fontsource-variable/inter`, latin subset, `font-display: swap`) for all UI/body/heading text. Secondary: **JetBrains Mono** (variable, self-hosted via `@fontsource-variable/jetbrains-mono`) reserved for tech-stack tag chips and code references — not used on any Phase 1 surface, declared now so Phase 3 case studies don't introduce an undeclared third typeface. |

**shadcn decision rationale:** Not initialized. This project's own stack research (CLAUDE.md "What NOT to Use") explicitly excludes heavy client component libraries because they undermine Astro's zero-JS-by-default performance goal, and 01-RESEARCH.md's architecture already specifies plain `.astro` components for the nav/footer shell — there are no complex interactive primitives (dialogs, comboboxes, etc.) in this phase's scope that would justify a component library. Self-hosted fonts (not Google Fonts CDN) chosen to avoid an external network request, supporting the project's own performance goals (POLISH-02) from day one.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing (nav link gaps, footer icon gaps) |
| md | 16px | Default element spacing (nav item padding, footer column padding) |
| lg | 24px | Section padding (nav bar vertical padding, footer top padding) |
| xl | 32px | Layout gaps (space between footer columns) |
| 2xl | 48px | Major section breaks (space above/below placeholder home content) |
| 3xl | 64px | Page-level spacing (top-level page margin on desktop) |

Exceptions:
- Footer social icon links (LinkedIn/GitHub/Kaggle/Email): visual icon may render at 20–24px, but the clickable/tappable area must be a minimum **44×44px** touch target (WCAG 2.5.5) — pad the `<a>` element, don't rely on icon size alone.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 32px | 600 (semibold) | 1.2 |

Exactly 2 weights declared (400, 600) — do not introduce a third weight in this phase.

**Role usage for Phase 1 surfaces:**
- **Display (32px/600):** Placeholder home page H1.
- **Heading (20px/600):** Nav brand/site-name text (short form: "Gerson Lopes"), 404 page heading.
- **Label (14px/600):** Nav links, footer links/column headings — semibold for legibility against the dark background at small size.
- **Body (16px/400):** Any paragraph copy (placeholder home subtext, 404 body copy).

## Color

Theme: **dark tech**, single theme, no light/dark toggle (D-07 — locked).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0e14` | Page background (`<body>`), the base surface for all pages |
| Secondary (30%) | `#111827` | Nav bar background, footer background — a slightly lighter elevation than the page background so the persistent shell reads as a distinct layer |
| Accent (10%) | `#22d3ee` | Reserved for: link hover/focus color, active/current-page nav indicator (underline or text color), footer icon hover state, focus-visible outline ring |
| Destructive | `#ef4444` | Not used anywhere in Phase 1 (no destructive actions exist yet) — reserved for future confirmation dialogs/delete actions in later phases |

Accent reserved for: nav link hover/focus state, current-page nav indicator, footer social-icon hover state, and the `:focus-visible` outline ring on all interactive elements. Never apply accent to large surface fills, body text, or as a default (non-hover/non-active) link color.

**Supplementary tokens** (not part of the 60/30/10 split but required to implement it correctly):
| Token | Value | Usage |
|-------|-------|-------|
| Foreground (text) | `#e2e8f0` | Default body/heading text color on the dominant/secondary surfaces |
| Muted foreground | `#94a3b8` | Footer secondary text (e.g. copyright line), de-emphasized labels |
| Border | `#1e293b` | Hairline border between nav/footer and page content, card/divider borders in later phases |
